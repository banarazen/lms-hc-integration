Unknown caller leaves voicemail → Smart Bot creates lead → Agent responds via LMS → Conversation syncs to HyperConnect

Guest (unknown number) leaves voicemail in HyperConnect
Guest → HyperConnect
2
Voicemail Smart Bot processes voicemail, creates guest indicator with lead flag in DB
HyperConnect → Smart Bot → Database
3
New Lead created in AI Lead Manager (source: voicemail). Timeline event stored.
Smart Bot → LMS → Database
4
SignalR pushes "New lead created" event to HyperConnect timeline. Lead badge appears.
Database → SignalR → HyperConnect
5
Sales agent sends SMS from LMS to lead. Message stored in lead timeline.
Agent → LMS → Database
6
SMS syncs to HyperConnect via SignalR. Appears in person's timeline with "via LMS" tag.
Database → SignalR → HyperConnect (bidirectional sync)
7
Guest replies via SMS. Reply appears in HyperConnect AND all active lead timelines in LMS.
Guest → HyperConnect → SignalR → LMS (all active leads)

📋 Flow 2: Existing Guest Creates New Lead → Dual Timeline Behavior

2-year guest "Anurag Duke" inquires about Botox → Agent creates lead from HC → Lead-specific events stay in LMS; communications sync
1
Existing guest "Anurag Duke" (2-year history) inquires about Botox treatment in HyperConnect
Guest → HyperConnect
2
Agent creates lead for Anurag from HyperConnect. Lead linked to existing guest ID. Lead flag added.
Agent in HC → LMS → Database
3
"New lead created" event appears in HC timeline. Lead badge added to Anurag's entry.
SignalR → HyperConnect
4
⚠ HyperConnect timeline: Shows 2-year history + "New lead created" event. Full person context preserved.
⚠ LMS timeline: Fresh timeline for Botox lead only. No 2-year history.
Different paradigms: Person-centric (HC) vs Lead-centric (LMS)
5
Agent adds Task "Follow up on Botox pricing" in LMS.
LMS only — does NOT sync to HC (lead-specific event)
6
Agent sends SMS "Here's the Botox pricing info" from LMS.
LMS → SignalR → HyperConnect (appears in Anurag's 2-year timeline)
💡 Key Design Principle: Communications (SMS, calls, voicemails) sync bidirectionally. Lead-specific management events (tasks, notes, stage changes) remain in LMS only. This preserves distinct value propositions of both products.

📋 Flow 3: AI Call Scheduling → Rule Validation → Execution → Result Logging

Agent selects leads → Schedules AI calls → Business hours validated → AI executes calls → Results logged in both systems
1
Sales agent selects lead(s) in LMS, clicks "AI Call", fills prompt & context in scheduling modal.
Agent → LMS UI
2
Business Hours Rules Engine validates schedule against state/region rules.
✔ If within business hours: Approved
❌ If outside: Auto-adjusts to next valid time slot (e.g., 9:00 AM next business day)
Scheduler → Rules Engine → Scheduler
3
"AI call scheduled" event added to LMS timeline. Agent notified of confirmed time.
Database → SignalR → LMS
4
At scheduled time, AI engine executes call. Can run 100+ parallel calls simultaneously (bulk mode at 7:30 AM).
AI Calling Engine → Lead Phone Numbers
5
Call results logged: Answered/Not answered, duration, sentiment score, key insights, recording link.
AI Engine → Database
6
"AI call completed" event appears in LMS timeline AND syncs to HyperConnect person timeline.
Database → SignalR → LMS + HyperConnect

📋 Flow 4: Lead Conversion → Badge Removal → Guest Transition

Lead marked as Won → Confirmation → Badge removed in HC → Timeline archived in LMS → Guest profile continues
1
Sales manager clicks "Convert Lead" in LMS (or sets sales stage to "Won").
Agent → LMS UI
2
Confirmation dialog shown: "Mark this lead as converted? This will remove the lead indicator in HyperConnect."
LMS UI
3
Lead flag removed in database. SignalR pushes real-time update.
LMS → Database → SignalR
4
HyperConnect: Lead badge removed immediately. "Lead converted" event added to timeline. Entry becomes regular guest.
SignalR → HyperConnect
5
LMS: Lead status changes to "Converted" (read-only). Timeline archived but accessible. If person has other leads, badge count updates (e.g., "2 Leads" → "1 Lead").
LMS UI update
6
Guest profile continues: All communications from before and after conversion visible in HyperConnect. Full continuity preserved.
HyperConnect (person-centric paradigm)

📋 Timeline Event Sync Rules

What syncs where and in which direction
Event Type Visible in LMS Visible in HC Sync Direction Notes
SMS Inbound ✔ Yes ✔ Yes ↔ Bidirectional Appears in all active leads for person
SMS Outbound ✔ Yes ✔ Yes ↔ Bidirectional "via LMS" tag shown in HC
Manual Call ✔ Yes ✔ Yes ↔ Bidirectional Includes duration, outcome
AI Call ✔ Yes ✔ Yes ↔ Bidirectional Includes sentiment, recording
Voicemail ✔ Yes ✔ Yes ↔ Bidirectional May trigger lead creation
Task ✔ Yes ❌ No — None Lead-specific management event
Note ✔ Yes ❌ No — None Lead-specific management event
Private Note ✔ Yes (private) ❌ No — None Internal-only, never synced
Lead Stage Change ✔ Yes ❌ No — None Lead-specific management event
New Lead Created ✔ Yes ✔ Event marker → LMS to HC Clickable deep link to LMS
Lead Converted ✔ Yes ✔ Event marker ↔ Bidirectional Triggers badge removal in HC
Appointment Booked ✔ Yes ✔ Yes ↔ Bidirectional Guest event, visible in both
